# Support

## Channels

- **Issues & bug reports:** <https://github.com/storyofcarl/jex-library/issues>
- **Security reports:** see [`SECURITY.md`](./SECURITY.md) — do not file security issues publicly.
- **Commercial & licensing inquiries:** open an issue tagged `licensing`.

## Support tiers

| Tier | Audience | Channel | First-response target |
| --- | --- | --- | --- |
| Community | Evaluation & open users | GitHub issues | Best effort |
| Standard | Commercial license holders | GitHub issues (priority queue) | 2 business days |
| Priority | Commercial license + priority add-on | Priority queue + escalation | 1 business day |

Business days are Monday–Friday, excluding public holidays. First-response targets cover
acknowledgement and triage, not resolution time.

## Filing a good bug report

Include:

- the package and version (e.g. `@jects/grid@0.8.0`);
- framework and version (Vanilla / React / Vue / Angular / Web Components);
- browser and OS;
- a minimal reproduction (a single-file repro or a link to one of the `examples/` apps with changes);
- expected vs. actual behavior, and any console errors.

## Severity

| Severity | Definition | Standard target | Priority target |
| --- | --- | --- | --- |
| S1 | Production down / data loss, no workaround | 1 business day | 4 business hours |
| S2 | Major feature broken, workaround exists | 2 business days | 1 business day |
| S3 | Minor / cosmetic | Next release cycle | Next release cycle |
| S4 | Question / enhancement | Best effort | Best effort |
